# BarLeads Newsletter — spec (phase 1: brand + scheduling groundwork)

## Cadence
Two sends per month, per venue:
- **Edition A** — the **last Wednesday of each month**.
- **Edition B** — exactly **two weeks after** Edition A (lands mid-following-month).

## Event window (~37 days)
Each email includes the venue's upcoming events within roughly 37 days of send:
- **Edition A** (last Wed of month M): all of month M+1's events (plus any still-future
  stragglers in the last days of M).
- **Edition B** (M+1, ~day 8-14): re-iterates month M+1 events still in the future, plus
  events up to ~2 weeks into month M+2.

## Ticketed events
Events (occurrence or custom) carry an optional `ticket` URL. In the newsletter, an
event with a ticket link gets the brand-styled CTA button ("Get Tickets") linking to it;
events without one get no button.

## Brand guide
- Generated by Claude by scraping the venue's website when `website` is set (by the
  venue via the 🌐 Website button, or by Kevin in /admin). Until generated, `brand` holds
  `{status:"pending", source_url}` — the admin panel shows it queued.
- Guide contents: palette (hex), email-safe font stack, dark/light mode, button styles,
  corner radius, motifs, aesthetic description, email layout rules, logo URL.
- Goal: emails look like they come directly from the venue's brand, not from a template.

## Imagery
- Kevin shoots each venue; assets live in a Dropbox folder linked per-venue in /admin
  (`dropbox_url`). Monthly newsletter visuals get generated with Higgsfield from those
  assets. Faces: avoid focusing on them; blur/overlay graphics when present (TBD).
